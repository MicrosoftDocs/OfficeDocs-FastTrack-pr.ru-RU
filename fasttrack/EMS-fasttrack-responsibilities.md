# <a name="fasttrack-responsibilities"></a>Обязанности специалистов FastTrack

Во время входящей миграции FastTrack имеет следующие обязанности.

## <a name="general"></a>Общие

-   Предоставление помощи по удаленной поддержке для действий, необходимых для выполнения действий по настройке, перечисленных в подробных описаниях этапов.

-   Предоставьте доступную документацию, программные средства и консоль администрирования, которые помогут вам уменьшить или устранить задачи настройки.

## <a name="initiate-phase"></a>Начальная фаза

-   Приступите к работе, чтобы начать присоединение.

-   Мы определим, какие поддерживаемые службы вам нужно перенести.

## <a name="assess-phase"></a>Фаза оценки

-   Мы предоставим вам административный обзор.

-   Вы можете получить помощь по указанным ниже вопросам.

    -   Требования к DNS, сети и инфраструктуре.

    -   Потребности клиента (веб-браузер, клиентская операционная система и службы).

    -   Удостоверения пользователей и подготовка.

    -   Включение соответствующих услуг, приобретенных и определенных для входящей миграции.

-   Мы составим график действий по устранению проблем.

-   Предоставьте контрольный список исправления для Intune и Azure AD Premium.

## <a name="remediate-phase"></a>Фаза исправления

-   Применяйте звонки с вами в соответствии с согласованным расписанием, чтобы проанализировать ход выполнения действий по устранению, например, перед переходом в облачную службу Майкрософт.

## <a name="enable-phase"></a>Фаза включения
Вы можете получить помощь по указанным ниже вопросам.

-   Активация клиента или подписки Microsoft Online Service.

-   настройке протоколов TCP/IP и портов брандмауэра;

-   настройке DNS для поддерживаемых служб;

-   Проверка возможности подключения к службам Microsoft Online.

-   Для среды с одним лесом:

    -   Установка сервера синхронизации каталогов между доменными службами Active Directory (AD DS) и подходящими службами Microsoft Online (только в случае необходимости).

    -   Настройка управляемой проверки подлинности (синхронизация хэша паролей или сквозная проверка подлинности) с помощью средства Azure Active Directory Connect. (при необходимости — только руководство).

        > [!NOTE]
        > Разработка и реализация расширений настраиваемых правил выходят за рамки области.

-   Для одного леса, когда цель — федеративные удостоверения: Установка и Настройка служб федерации Active Directory (AD FS) для проверки подлинности локального домена с Intune в отказоустойчивой конфигурации с одним сайтом, если это необходимо.

    > [!NOTE]
    > Для всех конфигураций с несколькими лесами развертывания AD FS выходят из области.

-   Тестирование функциональности единого входа (SSO), если она развернута.

### <a name="enable-phase---microsoft-azure-active-directory-premium"></a>Фаза включения — Microsoft Azure Active Directory Premium

Вы можете получить помощь по указанным ниже вопросам.

- Активация клиента Azure AD Premium.

- Настройка портов брандмауэра.

- настройке DNS для поддерживаемых служб;

- Проверка возможности подключения к службам Azure AD Premium.

- Для среды с одним лесом:

  -   Установка синхронизации каталогов между доменными службами Active Directory (AD DS) и Azure AD Connect, если это необходимо.

  -   Настройка метода проверки подлинности (синхронизации хэша паролей или сквозной проверки подлинности) с помощью средства Azure AD Connect.

- Для среды с несколькими лесами:

  -   Установка синхронизации с Azure AD Connect, Настройка для нескольких сценариев леса.
- Для сред с одним или несколькими лесами:
  - Настройка сквозной проверки подлинности Azure Active Directory (при необходимости).
  - Настройка эффективного единого входа Azure Active Directory (при необходимости).
    > [!NOTE]
    > Проверка подлинности на основе Azure Active Directory для сред с несколькими лесами поддерживается при наличии доверия лесов между лесами Active Directory и при правильной настройке маршрутизации суффиксов имен. Дополнительные агенты можно установить на нескольких локальных серверах, чтобы обеспечить высокую доступность для запросов на вход.

  - Дополнительные сведения см. в статье [Azure Active Directory сквозная проверка подлинности: краткое](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-quick-start#step-1-check-prerequisites) руководство и [Azure Active Directory, единый вход: быстрый запуск](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-sso-quick-start#step-1-check-prerequisites).
  - Дополнительные сведения об ограничениях сквозной проверки подлинности см. в статье [Текущие ограничения сквозной проверки подлинности Azure Active Directory](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-current-limitations).
  - Дополнительные сведения о проблемах, связанных с простым единым входом, см. в статье [Устранение неполадок с простым единым входом Azure Active Directory](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-troubleshoot-sso).

      > [!NOTE]
      > Синхронизация хэша паролей и обратная запись пароля поддерживают несколько лесов. Однако другие сценарии обратной записи не поддерживаются.

  - Настройка синхронизации между локальными лесами Active Directory и каталогом Microsoft Azure Active Directory Premium (Azure Active Directory).

    > [!NOTE]
    > Разработка и реализация расширений настраиваемых правил выходят за рамки области.

- Для одного леса, когда целевым объектом являются федеративные удостоверения:

  -   Установка и настройка AD FS для проверки подлинности локального домена с помощью Azure AD Premium в отказоустойчивой конфигурации с одним сайтом (при необходимости).

  > [!NOTE]
  > Для всех конфигураций с несколькими лесами развертывания AD FS выходят из области.

- Тестирование функций единого входа (при развертывании).

### <a name="enable-phase---azure-ad-premium--with-azure-ad-connect-and-ad-fs"></a>Фаза включения — Azure AD Premium — с Azure AD Connect и AD FS

Предоставление рекомендаций по настройке:

- Подготовка пользователей, включая лицензирование.

- Синхронизация службы каталогов Azure AD Connect (с обратной записью пароля и синхронизацией хэша пароля).

  - Сброс пароля самообслуживания Azure Active Directory (SSPR).

  - Многофакторная проверка подлинности Azure.

  - До трех (3) или более программного обеспечения для интеграции приложений (SaaS) с единым входом (SSO) из службы [Azure Active Directory Marketplace](https://azure.microsoft.com/marketplace/active-directory/).

  - Автоматическая подготовка пользователей для предварительно интегрированных приложений SaaS, перечисленных в [списке учебник по интеграции приложений](https://docs.microsoft.com/en-us/azure/active-directory/saas-apps/tutorial-list), только для подготовки исходящей почты.

  - Настраиваемый экран входа в систему, включая логотип, текст и изображения.

  - Самостоятельная служба и динамические группы (группы).

  - Прокси-сервер приложения Azure Active Directory.

  - Работоспособность подключения Azure Active Directory.

  - Условный доступ к Azure Active Directory.

  - Условия использования Azure Active Directory.

  - Защита удостоверений Azure Active Directory.

  - Azure Active Directory: привилегированное управление удостоверениями.

  - Проверка доступа Azure Active Directory.

### <a name="enable-phase---intune"></a>Фаза включения — Intune

> [!IMPORTANT]
> FastTrack не поддерживает управление классическими компьютерами с Windows 10 с помощью Intune. FastTrack поддерживает только управление Windows 10 с помощью управления мобильными устройствами Intune (MDM).

Предоставьте **рекомендации** по:

-   Настройка удостоверений, которые будут использоваться Intune, с помощью локальной службы Active Directory или облачных удостоверений (Azure Active Directory).

-   Лицензирование конечных пользователей.

-   Добавление пользователей в подписку Intune, определение ролей ИТ, а также создание групп пользователей и устройств.

-   Настройка центра управления мобильными устройствами (MDM) в соответствии с потребностями управления, в том числе:

    -   Настройка Intune в качестве своего центра MDM.

    -   Настройка тестовых групп, которые будут использоваться для проверки политик управления MDM.

    -   Навигация на портале администрирования Intune для обнаружения информации о пользователях и устройствах.

    -   Настройка ролей Intune (оператор службы поддержки, администраторы и т. д.)

    -   Настройка политик управления MDM и таких служб, как:

        -   Развертывание приложений для каждой поддерживаемой платформы с помощью веб-ссылок, MSI и/или глубоких ссылок.

        -   Развертывание Office профессиональный плюс на устройствах с Windows 10.

        -   Volume Purchase Programs for App Deployment, в том числе VPP Apple, Магазин Windows для бизнеса и игра Google для рабочего хранилища.

        -   Развертывание электронной почты, беспроводных сетей и профилей VPN, если у вас есть центр сертификации, сеть Wi-Fi или VPN в Организации.

        -   Настройка Microsoft Intune Exchange Connector (при необходимости).

        -   Профили конфигурации устройств для поддерживаемых платформ устройств.

    -   Настройка политик условного доступа.

    -   Настройка и развертывание политик защиты приложений Intune для каждой поддерживаемой платформы.

    -   Подготовка бизнес-приложений (LOB) для политик защиты приложений Intune с указанием доступных параметров.

    -   Регистрация устройств каждой поддерживаемой платформы в Intune или Configuration Manager с помощью службы Microsoft Intune.

    -   Подключение к хранилищу данных Intune.

    -   Интеграция Intune с:
        -   Средство просмотра Team Viewer для удаленного помощника (необходимо подписку на Team Viewer).

        -   Решения для защиты от угроз для мобильных устройств (требуются подписки на мобильные решения для защиты от угроз).

        -   Решения для управления расходами на телекоммуникации (необходимо указать подписку на решение для управления расходами на связи).

        -   Advanced Threat Protection в Защитнике Windows (требуются лицензии Windows или Microsoft 365 для вододействия).

    -   Настройка обновлений программного обеспечения для поддерживаемых поддерживаемых платформ.

    -   Материалы по планированию внедрения пользователей.

- Настройка автопилота Windows:

    - Настройка и настройка Microsoft Intune для Windows "Автопилот".

    - Настройка динамических групп Azure AD

    - Добавьте фирменную символику компании в Azure AD.

    - Создайте и назначьте устройства для профилей Windows в режиме автопилота (например, профиль Windows автопилота, который запрещает создание учетной записи локального администратора).

    - Настройка готового к работе (OOBE) в соответствии с требованиями Организации.

    - Настройка автоматической регистрации MDM в Azure AD и Intune.

    > [!NOTE]
    > Настройка Windows автопилота за пределами Intune выходит за область действия FastTrack.

### <a name="enable-phase---co-management"></a>Фаза включения — совместное управление

Вы можете получить помощь по указанным ниже вопросам.

-   Лицензирование конечных пользователей.

-   Добавление пользователей в подписку Intune, определение ролей ИТ-администраторов и создание групп пользователей и устройств (если Intune не установлено).

-   Настройка Azure Active Directory для автоматической регистрации MDM.

-   Настройка гибридного подключения Azure Active Directory.

-   Настройка шлюза облачного управления.

-   Добавление пользователей в подписку Intune, определение ролей ИТ, а также создание групп пользователей и устройств.

-   Подготовка Intune (если Intune не установлен):

    -   Настройка центра управления мобильными устройствами (MDM) в соответствии с потребностями управления, в том числе:

    -   Настройка Intune в качестве своего центра MDM.

    -   Настройка тестовых групп, которые будут использоваться для проверки политик управления MDM.

    -   Навигация на портале администрирования Intune для обнаружения информации о пользователях и устройствах.

    -   Настройка ролей Intune (оператор службы поддержки, администраторы и т. д.)

    -   Настройка и развертывание политик защиты приложений Intune для каждой поддерживаемой платформы.

    -   Регистрация устройств с Windows 10 в Intune.

- Включение функции совместного управления в консоли Configuration Manager.

- Переключить рабочие нагрузки на Intune.

- Отслеживайте действие по совместному управлению в вашей среде.

### <a name="enable-phase--azure-information-protection"></a>Фаза включения — Azure Information Protection

Предоставьте рекомендации по: 

- Клиентам автоматически классифицировать и помечать информацию в приложениях Office (например, Word, PowerPoint, Excel и Outlook), работающих в Windows, и с помощью клиента Azure Information Protection. 
- Файлы в REST с помощью сканера Azure Information Protection.
- Сообщения электронной почты на пути с помощью правил для почтовых ящиков Exchange Online. 

Поддержка также предоставляется пользователям, которые хотят применять защиту с помощью службы управления правами Microsoft Azure (Azure RMS), службы шифрования сообщений Office 365 (OME) и защиты от потери данных (DLP).

- Активируйте и настройте свой клиент.
- Создайте и настройте метки и политики.
- Примените защиту информации к документам.
