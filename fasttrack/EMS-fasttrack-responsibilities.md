# <a name="fasttrack-responsibilities"></a>Обязанности специалистов FastTrack

В процессе входящей миграции специалисты FastTrack отвечают за указанные ниже аспекты.

## <a name="general"></a>Общие

-   Предоставление удаленной поддержки по необходимым задачам настройки, перечисленным в подробных описаниях фаз.

-   Предоставление доступной документации, программных средств и консолей администрирования для уменьшения объема задач по настройке или полного избавления от них.

## <a name="initiate-phase"></a>Начальная фаза

-   Мы поможем начать перенос в облачную среду.

-   Мы определим, какие поддерживаемые службы вам нужно перенести.

## <a name="assess-phase"></a>Фаза оценки

-   Мы предоставим вам административный обзор.

-   Вы можете получить помощь по указанным ниже вопросам.

    -   Требования к DNS, сети и инфраструктуре.

    -   Требования клиента к веб-браузеру, операционной системе и службам.

    -   Удостоверения пользователей и подготовка.

    -   Включение поддерживаемых служб, приобретенных и определенных для выполнения входящей миграции.

-   Мы составим график действий по устранению проблем.

-   Мы предоставим контрольный список устранения проблем для Intune и Azure AD Premium.

## <a name="remediate-phase"></a>Фаза устранения ошибок

-   Мы будем проводить с вами сеансы конференц-связи по заранее оговоренному расписанию для обсуждения хода устранения проблем, например инструктировать по предварительным требованиям к установке перед подключением к облачной службе Майкрософт.

## <a name="enable-phase"></a>Фаза включения
Вы можете получить помощь по указанным ниже вопросам.

-   Активация клиента или подписки веб-службы Майкрософт.

-   Настройка протоколов TCP/IP и портов брандмауэра.

-   Настройка DNS для поддерживаемых служб.

-   Проверка возможности подключения к веб-службам Майкрософт.

-   Для среды с одним лесом:

    -   Установка сервера синхронизации каталогов между доменными службами Active Directory (AD DS) и поддерживаемыми веб-службами Майкрософт (только инструкции, если требуется).

    -   Настройка управляемой проверки подлинности (синхронизация хэша паролей или сквозная проверка подлинности) с помощью средства Azure Active Directory Connect. (только инструкции, если требуется).

        > [!NOTE]
        > Разработка и реализация расширений пользовательских правил не рассматриваются.

-   Для одного леса, если целью являются федеративные удостоверения: установка и настройка служб федерации Active Directory (AD FS) для проверки подлинности локального домена в Intune при отказоустойчивой конфигурации с одним сайтом, если это необходимо.

    > [!NOTE]
    > Для любых конфигураций с несколькими лесами услуги по развертыванию AD FS не предусмотрены.

-   Тестирование функций службы единого входа (SSO), если она развернута.

### <a name="enable-phase---microsoft-azure-active-directory-premium"></a>Фаза включения — Microsoft Azure Active Directory Premium

Вы можете получить помощь по указанным ниже вопросам.

- Активация клиента Azure AD Premium.

- Настройка портов брандмауэра.

- Настройка DNS для поддерживаемых служб.

- Проверка возможности подключения к службам Azure AD Premium.

- Для среды с одним лесом:

  -   Установка синхронизации каталогов между доменными службами Active Directory (AD DS) и Azure AD Connect, если это требуется.

  -   Настройка метода проверки подлинности (синхронизация хэша паролей или сквозная проверка подлинности) с помощью средства Azure AD Connect.

- Для сред с несколькими лесами:

  -   Установка синхронизации Azure AD Connect, настройка для сценариев с несколькими лесами.
- Для сред с одним или несколькими лесами:
  - Настройка сквозной проверки подлинности Azure Active Directory (при необходимости).
  - Настройка эффективного единого входа Azure Active Directory (при необходимости).
    > [!NOTE]
    > Сквозная проверка подлинности Azure Active Directory поддерживается в средах с несколькими лесами, если между лесами Active Directory существуют отношения доверия, а маршрутизация суффикса имен настроена правильно. Дополнительные агенты можно устанавливать на несколько локальных серверов, чтобы обеспечить высокую доступность для запросов на вход.

  - Дополнительные сведения см. в статье [Краткое руководство по сквозной проверке подлинности Azure Active Directory](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-quick-start#step-1-check-prerequisites) и [Простой единый вход Azure Active Directory — быстрый запуск](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-sso-quick-start#step-1-check-prerequisites).
  - Дополнительные сведения об ограничениях сквозной проверки подлинности см. в статье [Текущие ограничения сквозной проверки подлинности Azure Active Directory](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication-current-limitations).
  - Дополнительные сведения о проблемах, связанных с простым единым входом, см. в статье [Устранение неполадок с простым единым входом Azure Active Directory](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-troubleshoot-sso).

      > [!NOTE]
      > Синхронизация хэша паролей и обратная запись пароля поддерживают несколько лесов. Однако другие сценарии обратной записи не поддерживаются.

  - Настройка синхронизации между локальными лесами Active Directory и каталогом Microsoft Azure Active Directory Premium (Azure Active Directory).

    > [!NOTE]
    > Разработка и реализация расширений пользовательских правил не рассматриваются.

- Для одного леса, если целью являются федеративные удостоверения:

  -   Установка и настройка AD FS для проверки подлинности локального домена в Azure AD Premium при отказоустойчивой конфигурации с одним сайтом (если это необходимо).

  > [!NOTE]
  > Для любых конфигураций с несколькими лесами услуги по развертыванию AD FS не предусмотрены.

- Тестирование функций службы единого входа (если она развернута).

### <a name="enable-phase---azure-ad-premium---with-azure-ad-connect-and-ad-fs"></a>Этап включения — Azure AD Premium с Azure AD Connect и AD FS

Вы можете получить помощь по настройке следующих аспектов.

- Подготовка пользователей, включая лицензирование.

- Синхронизация службы каталогов Azure AD Connect (с обратной записью пароля и синхронизацией хэша паролей).

  - Самостоятельный сброс пароля (SSPR) Azure Active Directory

  - Многофакторная проверка подлинности Azure.

  - Интеграция до трех (3) или более приложений в виде услуги (SaaS) с единым входом из [Azure Active Directory Marketplace](https://azure.microsoft.com/marketplace/active-directory/).

  - Автоматическая подготовка пользователей для предварительно интегрированных приложений SaaS, перечисленных в [списке руководства по интеграции приложений](https://docs.microsoft.com/azure/active-directory/saas-apps/tutorial-list), ограниченных только исходящей подготовкой.

  - Настраиваемый экран входа в систему, включая логотип, текст и изображения.

  - Группы самообслуживания и динамические группы (группы).

  - Azure Active Directory Application Proxy.

  - Azure Active Directory Connect Health.

  - Условный доступ в Azure Active Directory.

  - Условия использования Azure Active Directory.

  - Защита идентификации Azure Active Directory.

  - Azure Active Directory Privileged Identity Management.

  - Проверки доступа Azure Active Directory.

  -   Защита пароля Azure Active Directory.

  -   Azure Active Directory B2B.

### <a name="enable-phase---intune"></a>Этап включения — Intune

> [!IMPORTANT]
> FastTrack не поддерживает классическое управление компьютером с Windows 10 при помощи Intune. FastTrack поддерживает управление Windows 10 только с помощью службы управления мобильными устройствами (MDM) Intune.

Вы можете получить помощь по указанным ниже вопросам.

-   Настройка удостоверений, используемых в Intune, с помощью локальной службы Active Directory или облачных удостоверений (Azure Active Directory).

-   Лицензирование конечных пользователей.

-   Добавление пользователей в подписку Intune, определение ролей ИТ-администраторов, а также создание групп пользователей и устройств.

-   Настройка центра управления мобильными устройствами (MDM) в соответствии с потребностями управления, в том числе:

    -   Настройка Intune в качестве центра MDM.

    -   Настройка тестовых групп, используемых для проверки политик управления MDM.

    -   Навигация на портале администрирования Intune для обнаружения информации о пользователях и устройствах.

    -   Настройка ролей Intune (оператор службы поддержки, администраторы и т. д.)

    -   Настройка политик управления MDM и таких служб, как:

        -   Развертывание приложений для каждой поддерживаемой платформы с помощью веб-ссылок, MSI и/или прямых ссылок.

        -   Развертывание Office профессиональный плюс на устройствах с Windows 10.

        -   Программы Volume Purchase Program для развертывания приложений, включая VPP Apple, Магазин Windows Store для бизнеса и Магазин Google Play for Work.

        -   Развертывание электронной почты, беспроводных сетей и профилей VPN, если в вашей организации используется центр сертификации, сеть Wi-Fi или VPN.

        -   Настройка соединителя Microsoft Intune с Exchange (при необходимости).

        -   Профили конфигурации устройств для поддерживаемых платформ устройств.

    -   Настройка политик условного доступа.

    -   Настройка и развертывание политик защиты приложений Intune для каждой поддерживаемой платформы.

    -   Подготовка бизнес-приложений (LOB) для политик защиты приложений Intune с указанием доступных параметров.

    -   Регистрация устройств каждой поддерживаемой платформы в Intune или диспетчере конфигураций с помощью службы Microsoft Intune.

    -   Подключение к хранилищу данных Intune.

    -   Интеграция Intune со следующими компонентами:
        -   Team Viewer для удаленной помощи (необходима подписка на Team Viewer).

        -   Решения партнеров Mobile Threat Defense (требуется подписка на решение партнера Mobile Threat Defense).

        -   Решения для управления затратами на телекоммуникации (требуется подписка на решение для управления затратами на телекоммуникации).

        -   Advanced Threat Protection в Microsoft Defender (требуется лицензия Windows E5 или Microsoft 365 E5).

    -   Настройка обновлений программного обеспечения для поддерживаемых платформ.

    -   Ресурсы по планированию внедрения среди пользователей.

- Настройка Windows Autopilot:

    - Конфигурация и настройка Microsoft Intune для Windows Autopilot.

    - Настройка динамических групп Azure AD

    - Добавление фирменной символики компании в Azure AD.

    - Создание и назначение устройств для профилей Windows Autopilot (например, для профиля Windows Autopilot, запрещающего создание учетной записи локального администратора).

    - Настройка интерфейса, запускаемого при первом включении (OOBE), в соответствии с требованиями организации.

    - Настройка автоматической регистрации MDM в Azure AD и Intune.

    > [!NOTE]
    > Настройка Windows Autopilot за пределами Intune недоступна в преимуществе FastTrack.

### <a name="enable-phase---cloud-attach"></a>Этап включения — подключение к облаку

Вы можете получить помощь по указанным ниже вопросам.

-   Лицензирование конечных пользователей.

-   Реализация подключения к облаку в консоли диспетчера конфигураций.

-   Добавление пользователей в подписку Intune, определение ролей ИТ-администраторов, а также создание групп пользователей и устройств (если служба Intune не установлена).

-   Настройка гибридного подключения Azure Active Directory.

-   Настройка Azure Active Directory для автоматической регистрации MDM.

-   Настройка шлюза облачного управления.

-   Добавление пользователей в подписку Intune, определение ролей ИТ-администраторов, а также создание групп пользователей и устройств.

-   Подготовка службы Intune для управления устройствами.

-   Настройка центра управления мобильными устройствами (MDM) для Intune.

-   Настройка тестовых групп, используемых для проверки политик управления MDM.

-   Навигация на портале администрирования Intune для обнаружения информации о пользователях и устройствах.

-   Настройка ролей Intune (оператор службы поддержки, администраторы и т. д.).

-   Регистрация устройств с Windows 10 в Intune.

-   Переключение рабочих нагрузок на управление с помощью Intune согласно требованиям.

### <a name="enable-phase--azure-information-protection"></a>Этап включения — Azure Information Protection

Вы можете получить помощь по указанным ниже вопросам. 

- Активация и настройка клиента пользователя.

- Создание и настройка меток и политик.

- Применение защиты данных к документам. 

- Автоматическая классификация и пометка сведений в приложениях Office (например, Word, PowerPoint, Excel и Outlook), работающих в Windows и использующих клиент Azure Information Protection.

- Использование неактивных файлов с помощью сканера Azure Information Protection.

- Отслеживание сообщений электронной почты в пути с помощью правил потока обработки почты Exchange Online.

Поддержка также предоставляется пользователям, которые хотят применять защиту с помощью служб Microsoft Azure Rights Management (Azure RMS), шифрования сообщений Office 365 (OME) и защиты от потери данных (DLP).

> **Хотите узнать больше?** См. статью [Enterprise Mobility + Security](https://www.microsoft.com/cloud-platform/enterprise-mobility).
